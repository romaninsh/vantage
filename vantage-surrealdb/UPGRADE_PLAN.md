# Vantage-SurrealDB 0.3 Upgrade Plan

## Completed

### Core Expression & Query Building ✅

- **`SurrealDB` struct** wraps `SurrealClient` with `Arc<Mutex>`.
- **`DataSource`** implemented (marker trait).
- **`ExprDataSource<AnySurrealType>`** implemented — `execute()` converts expressions to
  parameterized CBOR queries via `prepare_query`, `defer()` creates lazy closures.
- **`SurrealSelect`** implements `Selectable<AnySurrealType>` with full clause support (fields,
  WHERE, ORDER BY, GROUP BY, LIMIT/START, DISTINCT).
- **Result type transitions** via phantom types: `Rows` → `List`, `Rows` → `SingleRow`, `SingleRow`
  → `Single`, `List` → `Single`. Each has a typed `.get(&db)` method.
- **`surreal_expr!` macro** produces `Expression<AnySurrealType>` with automatic parameter
  conversion. Nested expressions flatten recursively via `ExpressionFlattener`.

### Type System ✅

- **`AnySurrealType`** generated by `vantage_type_system!` macro, wraps `ciborium::Value`.
- **`SurrealType` trait** implemented for: `bool`, `i8`–`u64`, `isize`/`usize`, `f32`/`f64`,
  `String`, `&str`, `char`, `Vec<T>`, `IndexMap<String, T>`, `Option<T>`, `Decimal`, `Thing`.
- **`Display` for `AnySurrealType`** renders human-readable values (integers, quoted strings, record
  IDs, arrays, maps, CBOR tags) — drives `preview()` output.
- **`Expressive<AnySurrealType>`** implemented for all scalar types + `AnySurrealType` itself, so
  scalars can be passed directly to `RefOperation` methods (`eq(25i64)`, `eq(true)`, etc).
- **`SurrealTypeVariants::from_cbor()`** detects all SurrealDB CBOR tags (datetime, duration, uuid,
  thing, decimal, geometry types, range).

### Expression Helpers ✅

- **`Identifier`** — auto-escaping field/table names (reserved keywords → `⟨name⟩`).
- **`Field`** — simple field reference, implements `Expressive<AnySurrealType>`.
- **`Thing`** — record ID (`table:id`), implements `SurrealType` (CBOR Tag 8) + `Expressive`.
- **`RefOperation`** trait — `eq`, `sub`, `contains`, `in_`, `rref`, `lref` on anything
  `Expressive<AnySurrealType>`. Accepts both scalars and expressions.
- **`Sum` / `Fx`** — `math::sum(...)` and generic function call wrappers.
- **`SurrealReturn`** — RETURN statement builder.
- **`SelectField`** / **`Target`** — SELECT clause and FROM clause components.

### Tests ✅

- 18 inline lib tests (select, operation, thing, types).
- 15 `select.rs` tests (query building, mock execution, type transitions, aggregation).
- 3 `return.rs` tests (RETURN statements, error handling — requires live SurrealDB).
- 6 `types.rs` tests (CBOR round-trip for strings, booleans, numbers, decimals, optionals — requires
  live SurrealDB).
- 7 doc-tests, all passing.

### Infrastructure ✅

- **`prepare_query`** converts `Expression<AnySurrealType>` → parameterized query string (`$_arg1`,
  `$_arg2`) + `IndexMap<String, AnySurrealType>` params, then converts params to CBOR.
- **Response parsing** in `execute()` extracts `result` from SurrealDB envelope
  `[{status: "OK", result: ...}]`, with proper error extraction for failed queries.
- **Recursive flattening** in `vantage-expressions` — nested expressions are fully flattened before
  parameter binding, supporting `Expressive` scalar wrappers.

## TableSource Implementation

### Phase A: Columns & Expression Factory ✅

Implemented in `surrealdb/impls/table_source.rs`. Associated types:

```rust
type Column<Type> = Column<Type>;          // from vantage-table (same as CSV)
type AnyType = AnySurrealType;
type Value = AnySurrealType;               // CBOR-backed values
type Id = Thing;                           // record IDs are Thing (table:id)
```

Completed methods:

- `create_column` → `Column::new(name)`
- `to_any_column` / `convert_any_column` → `Column::from_column()` (preserves original_type)
- `expr` → `Expression::new(template, params)`
- `search_table_expr` → placeholder (TODO: iterate searchable columns once TableLike exposes them)

No SurrealDB-specific column type needed — `Column<Type>` from vantage-table works directly.

### Additional traits (optional, unlocks more features)

All three traits share `build_select` internally. See `vantage-table/docs/interface-map.md` for full
cross-reference.

- **`SelectableDataSource`** — `select()` returns `SurrealSelect`. Already gives `Table::select()`,
  `get_count_query()`, `get_sum_query()`.
- **`TableQuerySource`** — `get_table_select_query()` returns a `SurrealSelect` for a table.
- **`TableExprSource`** — `get_table_count_expr()` / `get_table_max_expr()` / `get_table_min_expr()`
  / `get_table_sum_expr()` return `AssociatedExpression` for cross-table subqueries.

### Future: Mutation query objects

Currently `select()` returns a query object but write ops execute directly via `TableSource`.
Eventually, `SelectableDataSource` (or a new trait) could add `type Delete`, `type Update`,
`type Insert`, enabling `Table::delete_query()` / `update_query()` that share conditions with
`select()` via `build_select`. Low priority — mutation query inspection is rarely needed.

### Key decision: Record conversion

SurrealDB returns CBOR maps. Converting to `Record<AnySurrealType>` requires:

1. Parse CBOR `Map([(Text(k), v), ...])` → `IndexMap<String, AnySurrealType>`
2. This already works via `IndexMap::<String, AnySurrealType>::from_cbor()`
3. Wrap in `Record::from_indexmap()`

The `id` field in results needs special handling — it's a `Thing` (CBOR Tag 8), not a plain string.
`list_table_values` must extract IDs and return `IndexMap<Thing, Record<AnySurrealType>>`.
