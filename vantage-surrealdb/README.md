# Vantage SurrealDB

SurrealDB integration for the Vantage framework. Provides expression execution, query building
(`SurrealSelect`), type-safe CBOR protocol, and the foundation for full `Table<SurrealDB, E>`
support.

## Architecture

```text
┌─────────────────────────────────────────────────────┐
│                   vantage-surrealdb                 │
│                                                     │
│  SurrealSelect ──── Selectable<AnySurrealType>      │
│  SurrealDB     ──── ExprDataSource<AnySurrealType>  │
│                 ──── DataSource                     │
│  AnySurrealType ─── vantage_type_system! (CBOR)     │
│  RefOperation  ──── eq/sub/contains/in_/rref/lref   │
│                                                     │
│  surreal_expr! ──── Expression<AnySurrealType>      │
│  Identifier    ──── auto-escaping field names       │
│  Thing         ──── record IDs (Tag 8 CBOR)         │
│  Field         ──── simple field references         │
│  Sum / Fx      ──── math::sum / function calls      │
│  SurrealReturn ──── RETURN statement builder        │
└─────────────────────────────────────────────────────┘
         │                    │                │
    surreal-client    vantage-expressions   vantage-types
    (CBOR transport)  (Expression<T>,       (type_system!,
                       Selectable,           Entity,
                       ExprDataSource)       Record<T>)
```

## Quick Start

```rust
use surreal_client::SurrealConnection;
use vantage_surrealdb::{surreal_expr, surrealdb::SurrealDB};
use vantage_expressions::ExprDataSource;

let client = SurrealConnection::new()
    .url("cbor://localhost:8000/rpc")
    .namespace("test").database("test")
    .auth_root("root", "root")
    .connect().await?;

let db = SurrealDB::new(client);

let result = db.execute(&surreal_expr!("RETURN {}", 42)).await?;
println!("{}", result); // 42
```

## Expression Execution

`SurrealDB` implements `ExprDataSource<AnySurrealType>`. Expressions are built with `surreal_expr!`,
which produces `Expression<AnySurrealType>`. Parameters are automatically converted to CBOR and
bound as `$_arg1`, `$_arg2`, etc via `prepare_query`.

```rust
// Parameterized — safe from injection
let user = db.execute(&surreal_expr!(
    "CREATE user:test SET name = {}, age = {}",
    "Alice", 25
)).await?;

// Nested expressions flatten automatically
let condition = surreal_expr!("age > {}", 18);
let query = surreal_expr!("SELECT * FROM users WHERE {}", (condition));
let result = db.execute(&query).await?;
```

## Query Building with SurrealSelect

`SurrealSelect` implements `Selectable<AnySurrealType>` and supports result type transitions via
phantom types (`Rows` → `List` → `Single`, `Rows` → `SingleRow`).

```rust
use vantage_surrealdb::{select::SurrealSelect, select::field::Field, surreal_expr};
use vantage_expressions::Selectable;

// Basic query
let select = SurrealSelect::new()
    .with_source("product")
    .with_condition(Field::new("is_deleted").eq(false))
    .with_order(surreal_expr!("name"), true);
// SELECT * FROM product WHERE is_deleted = false ORDER BY name

// Aggregation
let count = SurrealSelect::new()
    .with_source("product")
    .with_condition(Field::new("active").eq(true))
    .as_count();
// RETURN count(SELECT VALUE id FROM product WHERE active = true)

// Type transitions
let single_name = SurrealSelect::new()
    .with_source("users")
    .with_condition(Field::new("id").eq("user:1"))
    .only_first_row()    // Rows → SingleRow (FROM ONLY)
    .only_column("name") // SingleRow → Single (SELECT VALUE)
    .get(&db).await?;    // returns AnySurrealType
```

## RefOperation — Comparison & Traversal

Any type implementing `Expressive<AnySurrealType>` gets `eq`, `sub`, `contains`, `in_`, `rref`,
`lref` methods. Scalars (`i64`, `bool`, `String`, `&str`) also implement
`Expressive<AnySurrealType>` and can be passed directly.

```rust
use vantage_surrealdb::{operation::RefOperation, thing::Thing, surreal_expr};

// Comparisons accept scalars or expressions
Field::new("age").eq(25i64);              // age = 25
Field::new("active").eq(true);            // active = true
Field::new("status").eq("completed");     // status = "completed"

// Graph traversal
Thing::new("bakery", "hill_valley").rref("owns", "product");
// bakery:hill_valley->owns->product
```

## Type System

`AnySurrealType` wraps `ciborium::Value` with type variant tracking. Generated by
`vantage_type_system!` macro from `vantage-types`. Supports `SurrealType` trait for `bool`,
`i8`–`u64`, `f32`/`f64`, `String`, `Vec<T>`, `IndexMap<String, T>`, `Option<T>`, `Decimal`, `Thing`,
plus CBOR tag detection for SurrealDB-specific types (DateTime, Duration, UUID, geometry, etc).

```rust
use vantage_surrealdb::types::{AnySurrealType, SurrealType, SurrealTypeVariants};

let val = AnySurrealType::new(42i64);
assert_eq!(val.type_variant(), Some(SurrealTypeVariants::Int));
assert_eq!(val.try_get::<i64>(), Some(42));
assert_eq!(format!("{}", val), "42"); // Display renders values, not type names
```

## Error Handling

All methods return `vantage_core::Result` with structured `VantageError`:

```rust
match db.execute(&surreal_expr!("SELECT =======")).await {
    Err(e) if e.to_string().contains("Parse error") => {
        // database syntax error
    }
    _ => {}
}
```
